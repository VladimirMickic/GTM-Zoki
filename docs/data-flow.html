How the data moves

Raw links in, ranked prospects out — one company at a time, on free-tier tools with Claude at the wheel.

◆ AI model ▢ tool / API ✓ pass ✕ reject the record grows at each step ↓
  1. 01

    Input

    ▢ serper

    Drop a list of URLs, or type a search — “companies that sell drones.” Claude auto-filters the messy results down to real manufacturers; no marketplaces, no listicles, no approval step.

    record = { company, website, source }

  2. 02

    Scrape

    ▢ crawl4ai

    Pull the site as clean markdown. If the primary chokes, it falls back automatically: Firecrawl → Scrapling → Apify → ScrapeGraphAI.

    record += raw_markdown

  3. 03

    Extract

    ◆ gpt-4o-mini

    The cheap workhorse reads the markdown and pulls structured fields — one place, so swapping scrapers never changes the schema.

    record += drone_models, drone_sizes, description, us_made/NDAA

  4. 04

    Fit & gate

    ◆ claude

    Claude scores the prospect 0–100 against company/ICP.md — does the airframe fit a case line, is it field-deployed, US-made, worth a budget? Hard disqualifiers drop instantly.

    record += fit_score, fit_reason

    ✓ score ≥ 40 → continue ✕ disqualified → denylist
  5. 05

    Contacts & enrich

    ◆ claude ▢ serper · company-research

    Passers only — no credits wasted on rejects. Find the right person (name, title, LinkedIn — no email yet), then pull profile + recent news, plus a LinkedIn and Reddit signal.

    record += contact_name, contact_title, contact_linkedin, key_news, linkedin, community_signals, buying_signals, outreach_angle

  6. 06

    Output

    ▢ gspread

    The finished record lands as one row in your Google Sheet, top prospects first — ready for you to work, and to grade. Contacts land on their own Contacts tab, one row per person.

    companyfitbest_casestatus
    Teal Drones87AV-Field▢ you grade
  7. ↻

    Learn

    ◆ claude

    Your grades in the status column and the reject bin feed a small feedback file. Claude reads it each run and proposes edits to the ICP and denylist — so run 02 starts smarter than run 01, and loops back to the top.

    feedback.jsonl → ICP.md + denylist → back to 01
